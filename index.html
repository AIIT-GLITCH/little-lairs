How It Works

1. Prompt

Send curated citation-demanding prompts to each model via standardized API calls. Same prompts, same conditions, every time.

2. Extract

Parse every URL from the response. Regex + heuristic extraction catches inline links, markdown references, and bare domains.

3. Verify

HTTP HEAD + GET each URL. Check status codes, redirects, DNS resolution, page content. Classify as alive, dead, or fabricated.

4. Classify

12-class taxonomy: alive, redirect, dead, timeout, DNS failure, fabricated domain, authority misuse, paywall, and more.

5. Score

Trust Score penalizes fabrication (60 pts) and dead links (30 pts). Models start at 100 and lose points for every bad anchor.

6. Rank

Best run per model goes on the leaderboard. Multiple runs allowed. Reproducible. All evidence artifacts archived.

Key Findings

6 models caught fabricating URLs

DeepSeek Chat, Gemma 2 27B, Mixtral 8x22B, Qwen 2.5 7B, Grok 3, and Nemotron 70B all produced at least one URL pointing to a domain or path that never existed. Nemotron leads with 2 fabrications across 83 anchors.

229 dead links across all models

Even models that don't fabricate still cite dead URLs at alarming rates. Llama 3.1 70B produced 18 dead links in a single run. The average model has 8.5 dead links per run.

6 models scored a perfect 100

Claude 3.5 Haiku, Claude 3.5 Sonnet, Command R+, Gemini 2.0 Flash, Mistral Large, and Qwen 2.5 72B all passed with zero fabrications and zero dead links (or no URLs cited).

About This Project

Little lAIrs is the public dashboard for AnchorForge, an epistemic benchmark that tests whether AI language models fabricate citations. Built by Rhet Dillard Wike at AIIT-THRESHOLD LLC in Council Hill, Oklahoma.

Every claim is verified. Every URL is checked. Every result is reproducible. The benchmark suite, scoring engine, and all evidence artifacts are open source.

Stack: FastAPI + Next.js 15 + PostgreSQL 16 + AnchorForge scoring engine
